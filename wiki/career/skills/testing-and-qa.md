# Testing and QA

Testing frameworks, strategies, and quality assurance practices across AI/ML and traditional software systems.

## AI/ML Testing

### DeepEval
- **Used at:** [[morningstar-software-engineer]]
- Model evaluation and monitoring for [[ai-document-processing]] and [[genai-classification]]
- Improved model reliability by **25%**
- Continuous monitoring of classification accuracy and drift detection

### PyTest
- **Used at:** [[morningstar-software-engineer]]
- Unit and integration testing for Python ML pipelines
- Paired with DeepEval for comprehensive model validation

## Traditional Testing

### JUnit
- **Used at:** [[bell-full-stack-developer]], [[virtusa-full-stack-developer]]
- Unit testing for Java microservices and banking transaction systems
- Test coverage for [[microservices-billing]] and [[banking-transaction-platform]]

### Selenium
- Browser automation and end-to-end testing
- UI testing for web applications

## API and E2E Testing

### Playwright
- **Used at:** [[morningstar-software-engineer]]
- API testing integrated with GitHub Actions
- Achieved **60% less regression** through automated test suites
- Cross-browser testing and scraping capabilities

## Testing Strategy by Domain

| Domain | Primary Tools | Focus |
|--------|--------------|-------|
| AI/ML pipelines | DeepEval + PyTest | Model accuracy, drift, reliability |
| Java microservices | JUnit | Unit, integration, contract tests |
| API testing | Playwright + GitHub Actions | Regression prevention |
| UI testing | Selenium, Playwright | E2E browser testing |
